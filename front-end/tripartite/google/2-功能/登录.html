编写 Google 登录逻辑 ​

在项目中创建一个登录组件：

vue
<!-- src/components/GoogleLogin.vue -->
<template>
  <div>
    <button @click="loginWithGoogle">使用 Google 登录</button>
    <div v-if="user">
      <p>欢迎，{{ user.name }}</p>
      <img :src="user.picture" alt="头像" />
    </div>
  </div>
</template>

<script>
export default {
  data() {
    return {
      user: null,
    }
  },
  methods: {
    loginWithGoogle() {
      if (!window.google) {
        console.error('Google SDK 未加载')
        return
      }

      window.google.accounts.id.initialize({
        client_id: '你的客户端ID.apps.googleusercontent.com',
        callback: this.handleCredentialResponse,
      })

      // 显示登录弹窗
      window.google.accounts.id.prompt((notification) => {
        if (
          notification.isNotDisplayed() ||
          notification.isSkippedMoment()
        ) {
          console.warn('用户取消登录或未显示')
        }
      })
    },
    handleCredentialResponse(response) {
      const jwt = response.credential
      const payload = JSON.parse(atob(jwt.split('.')[1])) // 解码 JWT，获取用户信息

      console.log('用户信息:', payload)
      this.user = {
        name: payload.name,
        email: payload.email,
        picture: payload.picture,
      }
    }
  }
}
</script>

使用组件 ​

编辑 App.vue 或创建页面引入：

vue
<template>
  <div id="app">
    <GoogleLogin />
  </div>
</template>

<script>
import GoogleLogin from './components/GoogleLogin.vue'

export default {
  components: {
    GoogleLogin
  }
}
</script>

最终效果 ​

运行项目：

bash
npm run serve

打开 http://localhost:8080，点击按钮即可弹出 Google 登录弹窗，成功后可显示用户信息。

附加说明 ​

  • response.credential 是一个 JWT（token），你可以发送给后端验证；

  • 后端可以用 google-auth-library 验证其有效性；

  • 若你想使用 Vue 插件方式封装登录，也可以用 vue-google-oauth2 实现。

💡 想要支持 One-Tap 登录？ ​

只需在页面上添加：

js
window.google.accounts.id.prompt()

它将自动检测用户并显示登录提示。

在以下代码中处理，解决方案有以下几种

用户取消登录处理 ​

✅ 解决方法 下面是用户如何手动重新启用 Google 登录（FedCM）权限的方法：

🌐 方法一：通过地址栏左边的“权限图标”重新授权 ​

  • 打开你的网站（例如：http://localhost:8080 或你的正式网址）
  • 点击地址栏左边的【🔒】小锁图标（或【ⓘ】图标）
  • 选择 "网站设置"（Site Settings）
  • 找到 "第三方登录"（Third-party sign-in）或 "FedCM"
  • 设置为 "允许（Allow）" 👉 如果是“已禁止”或“清除后禁止”，这里可以手动恢复。

⚙️ 方法二：通过 Chrome 设置页面重新授权 ​

  • 打开 Chrome 设置页面：chrome://settings/content
  • 向下找到 "第三方登录"（Federated login or sign-in）
  • 点击进入 → 你会看到被阻止的站点列表
  • 找到你的站点（如 localhost），将其设置为 "允许（Allow）"

🔄 方法三：清除浏览器缓存 + 重试登录 ​

如果你用的是 localhost 测试，有时拒绝之后缓存会留下设置。 步骤如下：

  • 打开 chrome://settings/clearBrowserData
  • 选择 Cookies 和其他站点数据
  • 清除与当前站点相关的记录（或只清除 localhost）
  • 重新打开页面 → 再次点击 Google 登录按钮

💡 补充建议：避免出现拒绝就“永久禁用”的 UX ​
